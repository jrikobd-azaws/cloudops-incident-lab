# Jira Workflow

## Purpose
This workflow is used to track operational work in the CloudOps Incident Lab in a structured, support-oriented way.

## Core issue types
- Incident
- Problem
- Change
- Service Request
- Task

## Incident lifecycle
1. **New**
   - Issue created after detection or report
   - Initial summary and scope recorded

2. **Investigating**
   - Technical checks begin
   - Impact and fault domain assessed
   - Evidence captured

3. **Escalated**
   - Additional team, owner, or dependency engaged
   - Clear handover context recorded

4. **Mitigated**
   - Immediate risk reduced
   - Service impact lowered or controlled
   - Monitoring continues

5. **Resolved**
   - Service restored or validation complete
   - Findings documented
   - Linked follow-up work identified

6. **Closed**
   - Final documentation complete
   - No further action required

## Problem lifecycle
1. New
2. Analysis
3. Root cause identified
4. Fix or recommendation defined
5. Closed

## Change lifecycle
1. Proposed
2. Approved
3. Implementing
4. Validation
5. Completed

## Service request lifecycle
1. Submitted
2. Reviewed
3. Approved
4. Fulfilled
5. Closed

## Workflow principle
The workflow is intentionally simple and operationally focused. It emphasizes triage quality, escalation timing, handovers, and documentation over administrative complexity.
